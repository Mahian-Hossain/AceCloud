The challenge

Parallel had achieved product-market fit quietly. 12,000 engineers found them via word-of-mouth and Hacker News threads — the site was never the acquisition channel. But with a Series A fundraise on the horizon, their investors made it clear: the site needed to match the product's reputation.

The brief had a hard constraint: developers are the audience, and developers are deeply sceptical of marketing. The site had to feel like it was built by engineers for engineers — not by a marketing team that had once met an engineer.

Our approach

We started with a week of user research — interviews with 12 power users from companies like Vercel, Resend, and Railway. The consistent signal: developers trust benchmarks, changelogs, and documentation quality. They distrust adjectives. The site needed to show, not tell.

The design direction: terminal-native. Every section earns its space with actual data — real benchmark numbers, real CLI output, real uptime stats. The dark theme isn't aesthetic — it's functional, matching the environment where developers actually work.

Built on Next.js 14 with MDX-powered docs and a custom changelog system. 98 Lighthouse. Shipped in 8 weeks including a complete documentation redesign.

Process

Developer empathy, encoded into every decision.

The design process started with research, not Figma. We needed to understand why developers trusted the product before we could design a site that conveyed that trust to new visitors.

Every design decision was pressure-tested against a single question: would a senior engineer at a company like Vercel find this credible?

Week 1
User research
12 interviews with power users. Identified trust signals: benchmarks, real CLI output, GitHub star counts, uptime data. Identified red flags: marketing adjectives, stock photography, vague promises.
Weeks 2–3
Information architecture
Restructured the site around the developer journey: problem awareness → curiosity → installation → activation. Each page has a single job in that funnel.
Weeks 3–5
Design system + build
Dark terminal aesthetic with Geist Mono for code and data. Purple accent (#5B5BD6) for interactive elements. Complete component library — nav, code blocks, benchmark tables, pricing, changelog. Next.js 14 + MDX docs.
Weeks 6–8
Docs redesign + launch
Full documentation redesign with search, versioned pages, and a community changelog system. Launched to 12,000 existing users with zero downtime migration.
